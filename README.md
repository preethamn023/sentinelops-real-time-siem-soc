# Overview

SentinelOPS is a Security Information and Event Management (SIEM) and Security Operations Center (SOC) monitoring platform designed to demonstrate real-world threat detection, log analysis, security monitoring, and incident response workflows within a controlled cybersecurity laboratory environment.

The platform integrates Wazuh, Python Flask, Linux-based infrastructure, and automated Telegram alerting to create an end-to-end security monitoring pipeline. Security events generated by monitored systems are collected, analyzed, correlated, and transformed into actionable alerts that can be viewed through a dashboard and delivered directly to analysts through mobile notifications.

Unlike many academic cybersecurity projects that rely on synthetic datasets or simulated alerts, SentinelOPS utilizes actual attack traffic generated within a virtualized lab environment. This approach enables practical observation of attack detection, event processing, and incident monitoring techniques commonly used in enterprise Security Operations Centers.

The primary objective of the project is to develop hands-on experience with SIEM architecture, security analytics, log management, threat detection, alert engineering, and SOC operations.

# System Architecture

The SentinelOPS architecture consists of four primary layers: attack simulation, monitored infrastructure, security analytics, and alert delivery.

The attack simulation layer generates security events using offensive security tools such as Hydra and Nmap. These attacks are executed from a Kali Linux virtual machine operating within an isolated laboratory environment.

The monitored infrastructure layer consists of an Ubuntu Server configured as the target system. Authentication logs, security events, and operating system telemetry generated by the server are continuously monitored by a Wazuh agent.

The security analytics layer is powered by Wazuh, which performs log collection, normalization, rule evaluation, event correlation, and threat detection. Security events identified by Wazuh are stored and processed for further analysis.

The alert delivery layer includes custom Python Flask services and Telegram integration. This layer processes security alerts, exposes data through APIs, and delivers real-time notifications to analysts while simultaneously providing data for dashboard visualization.

# Detection Engineering

SentinelOPS implements multiple detection mechanisms designed to identify suspicious and malicious activities within the monitored environment.

## SSH Brute Force Detection

The platform monitors repeated authentication failures occurring within a short period of time. When predefined thresholds are exceeded, Wazuh generates security alerts indicating potential password guessing activity.

## Network Reconnaissance Detection

Port scanning activities performed using Nmap generate observable network behavior that can be correlated with security events. These events provide visibility into reconnaissance activities commonly observed during the early stages of cyber attacks.

## Authentication Monitoring

The platform continuously monitors successful and failed authentication events. Suspicious authentication patterns can be identified and analyzed through centralized log management.

## Security Alert Processing

Detected events are automatically processed by custom Python services and transformed into structured alerts suitable for dashboard visualization and mobile notifications.

# Security Monitoring Workflow

The monitoring workflow implemented by SentinelOPS follows the same general principles used within enterprise Security Operations Centers.

1. An attack is launched from the Kali Linux attacker machine.
2. The target Ubuntu server generates security-related log entries.
3. Wazuh collects and analyzes security telemetry.
4. Detection rules evaluate incoming events.
5. Suspicious activities trigger security alerts.
6. Alert data is processed through Flask-based services.
7. Notifications are delivered through Telegram.
8. Security events are displayed within the monitoring dashboard.
